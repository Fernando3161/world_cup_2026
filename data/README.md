# Data Folders

This directory holds local source snapshots and generated data. Runtime frontend code must consume generated static files only and must not fetch live rating or match data.

- `raw/` - downloaded or manually placed raw source files.
- `manual/` - curated project data such as teams, aliases, source catalog, and bracket definitions.
- `snapshots/` - frozen rating and future official-result snapshots.
- `processed/` - normalized intermediate files generated by Python scripts.
- `frontend/` - generated frontend-ready data before it is copied or emitted to `frontend/public/data/`.

Historical calibration uses `raw/international_results/results.csv` and `raw/international_results/shootouts.csv`, plus metadata in `raw/international_results/source_metadata.json`. Raw historical files are used only by build-time Python scripts. They are not copied into the frontend public bundle.

Do not add fake production ratings or teams. Test fixtures should live under `tests/`.
